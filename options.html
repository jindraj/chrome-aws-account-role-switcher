AWS Role Switcher — Settings

AWS Config

Paste your ~/.aws/config content here. The extension supports SSO profiles and role chaining via source_profile.

Supported format: 
[profile base-sso]
sso_account_id = 123456789012
sso_role_name = AdministratorAccess
sso_region = eu-west-1
sso_start_url = https://myorg.awsapps.com/start

[profile child-role]
region = eu-west-1
role_arn = arn:aws:iam::999999999999:role/my-role
source_profile = base-sso
external_id = optional-external-id

[profile grandchild-role]
region = eu-west-1
role_arn = arn:aws:iam::888888888888:role/another-role
source_profile = child-role

How it works

  1. Configure your AWS profiles in the text area above (same format as ~/.aws/config).
  2. Click the extension icon to see all configured profiles.
  3. Click Switch on any profile.
  4. If the profile uses SSO, a browser tab opens for you to log in. After login, the extension automatically continues.
  5. For chained profiles, the extension automatically follows the source_profile chain using sts:AssumeRole.
  6. A new tab opens with the AWS console logged in as the target role.

Note: When using chained temporary credentials, AWS limits sessions to 1 hour maximum. SSO tokens are cached for up to 8 hours (or until expiry).
